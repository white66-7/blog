## 天气组件的实时动画适配
[地址](https://codepen.io/ste-vg/pen/GqaZbo)
这个组件我认为我做的非常之棒，每个动画转场做得非常丝滑，可是不能直接放进我的网站，因为原本这只是五个动画的演示。为了做适配，第一步是改变思路，将这个组件的功能改为实时天气显示。如何实现呢？我们不妨先来看看原版逻辑
```javascript
<div id="date">Monday 22 August</div>
<div class="temp">20<span>c</span></div>
<li><a id="button-snow" class="active"><i class="wi wi-snow"></i></a></li>
<li><a id="button-wind"><i class="wi wi-strong-wind"></i></a></li>
<li><a id="button-rain"><i class="wi wi-rain"></i></a></li>
<li><a id="button-thunder"><i class="wi wi-lightning"></i></a></li>
<li><a id="button-sun"><i class="wi wi-day-sunny"></i></a></li>
var weather = [
  { type: 'snow', name: 'Snow'}, 
  { type: 'wind', name: 'Windy'}, 
  { type: 'rain', name: 'Rain'}, 
  { type: 'thunder', name: 'Storms'},
  { type: 'sun', name: 'Sunny'}
];
for(var i = 0; i < weather.length; i++) {
  var w = weather[i];
  var b = $('#button-' + w.type);
  w.button = b;
  b.bind('click', w, changeWeather);
}
function changeWeather(weather) {
  if(weather.data) weather = weather.data; 
  reset();                        
  currentWeather = weather;        
  weather.button.addClass('active'); 
  //接下来是一堆过度动画
}
```
我们来看看他的逻辑，可以看到他的日期和温度是写死了的,没有改动的空间。另外他的动画转换是靠点击触发，在一个循环中，var b = $('#button-' + w.type)语句让每个b绑定html中每个按钮的id,接着给b绑定点击事件，点击(click)后触发changeWeather函数，对象是click后的变量w。至于w.button = b这个语句则是为了在weather.button.addClass('active')和 reset()函数中通过weather.button拿到其对应html中的<a>按钮给它添加与取消高亮模式。下一步我们来说说动画是怎么运行的，理解完这个就可以着手适配了。
```javascript       
requestAnimationFrame(tick);
function tick(){
tickCount++;
var check = tickCount % settings.renewCheck;
if(check)
{
	if(rain.length < settings.rainCount) makeRain();
	if(leafs.length < settings.leafCount) makeLeaf();
	if(snow.length < settings.snowCount) makeSnow();
}
for(var i = 0; i < clouds.length; i++)
{		
	if(currentWeather.type == 'sun'){
	if(clouds[i].offset > -(sizes.card.width * 1.5)) clouds[i].offset += settings.windSpeed / (i + 1);
	if(clouds[i].offset > sizes.card.width * 2.5) clouds[i].offset = -(sizes.card.width * 1.5);
	clouds[i].group.transform('t' + clouds[i].offset + ',' + 0);
	}
	else{	
	clouds[i].offset += settings.windSpeed / (i + 1);
	if(clouds[i].offset > sizes.card.width) clouds[i].offset = 0 + (clouds[i].offset - sizes.card.width);
	clouds[i].group.transform('t' + clouds[i].offset + ',' + 0);}
    }
	requestAnimationFrame(tick);
}
```
在网页刚加载时调用requestAnimationFrame(tick)函数，下一帧刷新是调用tick函数，tick函数中又做了一个无限递归，并且让三层云不停地水平移动，粒子补充检查并重新生成,这个稍微了解即可，其实是可以直接复制粘贴的。

首先要将日期温度还有天气换为响应式设计，这样在天气改变时显示也会修改
```javascript
<div class="temp">{{ temp }}<span>°C</span></div>
<div class="desc">{{ weatherDesc }}</div>
<div class="date">{{ formattedDate }}</div>
const weatherNames: Record<string, string> = {
  sun: 'Sunny', wind: 'Windy', rain: 'Rain', thunder: 'Storms', snow: 'Snow'
}
onMounted(async () => {
  await nextTick()
  initSVG()
  changeWeather('wind')
  tickId = requestAnimationFrame(tick)
  await fetchWeather()
})
async function fetchWeather() {
  try {
    const lat = 30.5833   // 纬度
    const lon = 114.2667  // 经度
    const res = await fetch(
      `https://api.open-meteo.com/v1/forecast?latitude=${lat}&longitude=${lon}&current_weather=true&timezone=auto`
    )
    const data = await res.json()
    const current = data.current_weather
    temp.value = String(Math.round(current.temperature))
    // 根据天气匹配动画
    const code = current.weathercode
    let type = 'wind'
    if (code === 0) type = 'sun'
    else if (code >= 1 && code <= 3) type = 'wind'
    else if (code >= 45 && code <= 48) type = 'wind'
    else if (code >= 51 && code <= 67) type = 'rain'
    else if (code >= 80 && code <= 82) type = 'rain'
    else if (code >= 95 && code <= 99) type = 'thunder'
    else if (code >= 71 && code <= 77) type = 'snow'
    weatherDesc.value = weatherNames[type] || 'Unknown'
    // 当前日期
    const now = new Date()
    formattedDate.value = now.toLocaleDateString('en-US', {
      weekday: 'long', day: 'numeric', month: 'long'
    })
    // 切换动画
    changeWeather(type)
  } catch (err) {
    console.error('获取天气失败', err)
    weatherDesc.value = 'Error'
    changeWeather('wind')
  }
}
```
在onmouted特性下，网页加载时即调用fetchWeather函数，在fetchWeather函数中，用fetch动作拿你的经纬度向Open-Meteo这个免费天气API发送请求，接着变量date将其转为json类型，得到的数据形式类似于下方数据
```json
"current_weather": {
    "temperature": 28.3,
    "windspeed": 12.5,
    "winddirection": 185,
    "weathercode": 3,
    "time": "2026-06-19T12:00"
  }
```
其中weathercode是天气代码表,time是位置国家的标准时间，temperature指温度

## 天气组件的大小和位置适配
要知道我这是一个多组件页面，我将盒子嵌套为四层，第一层就是主界面，第二层分为左侧盒子与右侧盒子，第三层是在右侧盒子中分为上盒子和下盒子，第四层在第三层的上盒子中我又分为左侧相册盒子还有天气盒子
```css
<div class="top-row">
  <div class="album-container">
  <ImageSlider :images="albumImages" />
  </div>
  <WeatherCard address="武汉" class="weather-card-comp" />
</div>
.top-row {
  display: flex;
  gap: 16px;             
  align-items: stretch;
}
.album-container {
  width: 500px;
  max-width: 100%;
  flex-shrink: 0;
}
.weather-card-comp {
  width: 280px;         
  flex-shrink: 0;  
  align-self: stretch; 
}
.weather-card {
  width: 100%;
  height: 100%;
  min-height: 300px;
  overflow: hidden;
  position: relative;
}
```
由于我导入的图片都是16:9分辨率，那么我设置相册展示的宽度后高度就自动确定了，的在这样的一个思路下，我只要设置好每个盒子的width就好了，并让大盒子的align-items: stretch(元素等高)抑或是天气组件的align-self: stretch(要和同级盒子等高)，这样高度就自动对齐了，并将两个盒子设置flex-shrink: 0;这样无论在窗口怎样缩小的变动下，他都会保持设置好的宽度。在具体的组件文件中，宽高完全继承，接着设置overflow: hidden;超出设置的边界即被裁切，然后设置设置position: relative为了给子元素绝对定位的锚点。

这样就基本实现了适配，老实说我写的并不全面，但是大概就差不多是这样的，由于对动画做的修改不具备泛用性，这里其实也没有必要具体说明了。
