## 组件元素与html文档分块
首先说说这个占据主要区域的是整张图片，在悬停之后照片会自动停止轮播，显现出左右箭头，点击后会跳到下一张，同样描述文本会从底部上升至卡片50%的高度。

为了达到这样的效果，先设立一个盒子，然后先分两个块，一个块用来放图片，另一块则用来放左右箭头。这个左右箭头，很显然是可以靠点击交互。所以给一个button外壳,@click绑定一个函数点击触发，span里的字符则是<，>这两个个符号。
```html
<button class="prev" @click="prevSlide">
<span>&#10094;</span>
</button>
```
接着则是主要的展示，在主要的展示还分了三层，第一层是可视窗口，第二层我把它叫做整张胶片，第三层则是图片本身与描述文本。解释一下为什么：第一层的可视窗口就像是取景器，用来具体显示胶片的某一小张而非整张胶片(那样会占很长的一个宽度显示多张图片)，而第二层是正常胶片，用来实现轮播的功能，第三层是显示具体的某一张图片和对应的描述的文本。

那么怎么显示图片并找到其对应的描述文本呢？我将其基本数据设计成一个数组，数组的元素是一个结构体，这个结构体包含两个元素，一个是图片地址，另一个是描述文本
```javascript
const albumImages = [
  { url: img1, description: `当时刚刚中考完特地换了张头像...` },
]
```
## 可视窗口设计的html实现
先设置一个名为slider-viewport的盒子，给他设置overflow为hidden的属性，接着再设置一个盒子
```html
<div class="slider-inner" 
:style="{ transform: `translateX(-${currentIndex * 100}%)` }">
```
这里直接给他的style的变化开关设置为水平移动函数，根据我的currentIndex(目前展示图片序号)显示对应的图片
接着在这个盒子中再设置一个轮播盒子来接收我的数据数组并利用
```html
<div class="slide" v-for="(image, idx) in images" :key="idx">
```
可以看到这个slide盒子，做了一个循环，其中image是子元素,也就是每个结构体，idx则相当于我们平常循环中的i。
接着又嵌套一个盒子，放两个平级的元素：图片和文本，图片和文本全靠直接调用元素image的url和description。而上个盒子的key则在这里用序号给每个盒子都加上了标记，这在以后有网页动态数据交互的时候会节省很多DOM内存开支，每次删改移动的时候都会看这个key是否存在，存在就不用再销毁创建，只要移动就好，是否但是现在由于是静态，固定的，基本没啥用。
## 轮播实现
为了实现轮播，就要重点注意这里的currentIndex(目前展示图片序号),让他随时间而改变。
如何实现呢？首先把他改成响应式数据类型，接着构造一整个函数链条
```javascript
const props = defineProps<{
  images: SlideImage[]
}>()
let timer: ReturnType<typeof setInterval> | null = null
const startAutoPlay = () => {
  if (props.images.length <= 1) return
  stopAutoPlay()
  timer = setInterval(nextSlide,4000)
}
const stopAutoPlay = () => {
  if (timer) {
    clearInterval(timer)
    timer = null
  }
}
onMounted(() => {
  startAutoPlay()
})
```
首先用defineProps声明从外界接受数组参数，这个就是我的基本数据了。接着来看这个startAutoPlay函数，我们先做一个边界检测，如果传进来的images没有图片，那就直接退出(显示为空)

接着设置一个防冲突机制，直接清除之前的任务，防止莫名的bug,一般来说加这个与否正常情况下没有影响。

最后设置一个定时器任务即setInterval，这两个参数分别为要实施的函数nextSlide，间隔时间4000毫秒。将这个任务赋值给timer，后续clearInterval就可以通过timer找到对应的计时器任务并且清除它。

用onMounted特性在网页开始的时候就自动加载startAutoPlay函数。
```javascript
const nextSlide = () => {
  currentIndex.value = (currentIndex.value + 1) % props.images.length
}
```
其中调用的nextSlide就修改了响应式数据的数值 :下一张通过求余每个数组的元素数量，从而实现循环。currentIndex修改后slider-inner这个盒子的transform属性更改，接着进行动画跳到下一张图片
## 配套的css文档
其实单单靠那些我的功能实现的看起来会很粗糙，加上css描述后就会变得精细。
```css
.slider-inner {
  display: flex;
  transition: transform 800ms cubic-bezier(0.77, 0, 0.175, 1);
}
```
先说三个最为基本的设置，slider-inner的flex排列会让所有的子元素排成一排，配合slide盒子的flex设置
```css
.slide {
  flex: 0 0 100%;
  aspect-ratio: 16 / 9;
  overflow: hidden;
}
```
三个属性对应flex-grow空间变大要不要拿多余空间，这里0为否1为是

flex-shrink空间不够时要不要缩水

flex-basis初始宽度占多少，这里是100%

剩余slide设置的aspect-ratio可以配合16:9分辨率的图片，overflow为hidden让卡片边缘变成圆角
接着slider-viewport的overflow为hidden就可以让我们看到强制占满一屏的slide而非整个胶片。
```css
.slider-viewport {
  overflow: hidden;
}
```
那么另外呢为了让转换变得更加顺滑，slider-inner还设置了一个transition，这样在transform变化就会有一个800毫秒的先快后慢的速度曲线。
这里又是一层嵌套
```css
.property-card {
  width: 100%;
  height: 100%;
  position: relative;
  border-radius: 16px;
  overflow: hidden;
}
```
这里property-card的position设置为relative就是给图片和文本的position定位提供依靠，接着就是图片和文本的css设置了。
```css
.property-image {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background-size: cover;
  background-position: center;
  background-repeat: no-repeat;
  transition: height 0.4s cubic-bezier(0.645, 0.045, 0.355, 1);
}
.property-description {
  position: absolute;
  bottom: 0;
  left: 0;
  width: 100%;
  height: 0;
  overflow: hidden;
  background-color: #fafafc;
  display: flex;
  align-items: center;
  justify-content: center;
  text-align: center;
  padding: 0;
  box-sizing: border-box;
  transition: height 0.4s cubic-bezier(0.645, 0.045, 0.355, 1),
              padding 0.4s cubic-bezier(0.645, 0.045, 0.355, 1);
}
```
两个子盒子位置设为absolute，接着配合left，top和botton属性可以让其紧贴父盒子左侧和上侧或是下侧
```css
.property-image {
  background-size: cover;
  background-position: center;
  background-repeat: no-repeat;
  transition: height 0.4s cubic-bezier(0.645, 0.045, 0.355, 1);
}
```
至于这三个属性，background-size的cover可以把背景图等比放大，直到图片的宽度或高度完全覆盖住整个盒子，比例不对则会被裁剪

background-position为center让图片完全居中

background-repeat: no-repeat是因为背景图默认情况下，如果尺寸小于盒子，它会像贴瓷砖一样在水平和垂直方向重复平铺，显示多张照片，这个设置可以让其只显示一张

transition属性则可以在height变化时，启动一个0.4s的动画
```css
.property-description {
  overflow: hidden;
  background-color: #fafafc;
  display: flex;
  align-items: center;
  justify-content: center;
  height: 0;
  padding: 0;
  box-sizing: border-box;
  transition: height 0.4s cubic-bezier(0.645, 0.045, 0.355, 1),
              padding 0.4s cubic-bezier(0.645, 0.045, 0.355, 1);
}
```
这里最为重要的是padding设置为0 :因为padding是内边距，尽管高度设置为0，如果padding不为0，那么图片依旧会被顶起来。那么为什么不设置margin也为0呢，这里原因是在absolute的位置设置下，margin默认就是0，要是你细心显式设置为0也没影响。

而设置flex的弹性布局后，接着把align-items为center，justify-content为center可以让子元素在中心位置，但老实说其实只有一个段落p子元素,完全可以用text-align设center代替。

transition属性则可以在height和padding变化时，启动一个0.4s的动画
### 重点hover设置
```css
.property-card:hover .property-image {
  height: 55%;
}
.property-card:hover .property-description {
  height: 45%;
  padding: 0.8em 1.2em;
}
```
鼠标悬停后两个子元素的height变化启动动画，不悬停就回归原来的初始设置。那么这里为什么又要给padding设置padding属性呢？

因为害怕文本过多过长会贴住上下左右边缘，非常难看，这样会带有一点呼吸感。反正你记住盒子里的文本肯定是要加padding属性的就对了。
